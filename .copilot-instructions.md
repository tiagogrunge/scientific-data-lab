# Scientific Data Lab - Development Guidelines

## Commit Message Format

Generate commit messages following this format:

```
[TYPE] Brief description (50 characters max)

- Detailed explanation of changes
- Use bullet points for multiple related changes
- Keep lines to 72 characters or fewer
- Be specific about what was changed and why
```

### Commit Types

- **feat**: New feature or functionality
- **fix**: Bug fix or issue resolution
- **refactor**: Code reorganization without feature changes
- **test**: Adding or updating tests
- **docs**: Documentation updates
- **style**: Code formatting, linting (no functional changes)
- **perf**: Performance improvements
- **chore**: Dependency updates, build changes

### Examples

```
[feat] Add CSV validation for malformed data

- Implement column count validation in load_csv()
- Raise ParserError for mismatched column counts
- Add test case for invalid CSV format
```

```
[fix] Resolve pandas 2.0.3 build dependency issue

- Update pandas to >=2.1.0 for Python 3.14 support
- Update numpy to >=1.26.0 compatible version
- Add pandas-stubs for type checking support
```

## Code Quality Standards

- Python version: >=3.10 (enforced by mypy)
- Type hints: Required for all function definitions
- Line length: 100 characters (enforced by ruff)
- Imports: Alphabetically sorted, grouped logically
- Tests: Required for all new features

## Testing

Run before committing:
```bash
pytest tests/ -v
mypy src/ tests/
ruff check src/ tests/
```
